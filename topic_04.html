Numeric Data Types

  • Numeric data types are of three types.
    1. Integers
    2. Floating Point Numbers
    3. Complex
  • Now, we cover these data types one by one.
  • These data types are the most used data types in science and a solid understanding will help write good computer programs.
  • They are not only important in science and engineering, but also in all other domains of work.

1. Integers

  • Integers are the whole numbers.
  • They are attributed to counting of objects.
  • Arithmetic operations are applicable and exact.
  • Examples of integers are 29, -40, and 11 etc.
  • Thousand, million, and billion separation are application with the use of underscore symbol e.g., 299,792,458 can be written as 299_792_458 in python.
  • The python returns int data type for integers.

2. Floating Point Numbers

  • Floating-point numbers are decimal point numbers.
  • They are attributed to measurements.
  • Arithmetic operations are applicable and accurate to 18 places of decimal in python.
  • Examples of integers are 3.14159, 8.314, and −2.00233184123 (electron g-factor) etc. They can also be represented in scientific notation e.g., 6.62607015×10-34 and 6.67430×10-11 etc. The python returns float data type for integers.

3. Complex

  • Complex data type consists of two part namely real part and imaginary part.
  • They are attributed to complex numbers.
  • Arithmetic operations are applicable as per mathematics rules for complex numbers.
  • Examples of complex numbers are 3+2ι3 + 2\iota3+2ι and 3−2ι3 - 2\iota3−2ι, and −4ι-4\iota−4ι etc.
  • In python, they are represented as 3+2j, 3-2j, and -4j etc.
  • The python returns complex data type for complex numbers.
  • The real part of a complex number can be called by the following line of code in python.
>>> (3.2 - 1.5j).real
  • The imaginary part of a complex number can be called by the following line of code in python.
>>> (3.2 - 1.5j).imag

Caution!

The symbol >>> is known as python prompt and you don’t need to type it. Always type the text after the python prompt symbol, for example, 3+2j only.

  • We know that complex numbers always exist in conjugate pairs.
  • Python has the facility to generate the conjugate pair of the given complex number automatically.
  • The conjugate pair can be called by the following line of code on python prompt.
>>> (3.-1.2j).conjugate()

Methods and Attributes

In calling real and imaginary parts, we used the keywords real and imag (no paranthesis after keyword), however, we used conjugate() keyword (paranthesis after keyword). The keywords real and imag are called the attributes while the keyword conjugate() is called a method. We will learn about these two terms in the upcoming lectures.

Assignment Operator

  • Consider the following mathematical equations
x=5x=x+3 x = 5\\ x = x + 3 x=5x=x+3
  • The above mathematical equations are wrong.
  • Suppose, we write the following two lines in a python program.
>>> x = 5
>>> x = x + 3
  • These two lines are valid in python programming.
  • In python programming, = is not the symbol for equality.
  • This symbol is knows as the assignment operator.
  • The job of the assignmet operator is to compute the right hand side of the assignment operator and assign it to the variable on the left hand side.
  • So, the first line assigns the value of 5 to variable x.
  • In the next line, python computes the right hand side which becomes 8 and this value is assigned to the variable x.
  • In other words, the value of x is updated.
  • So assignment operator assigns the results of right hand side to the variable on the left hand side.

Variables

  • In python, we can assign data (any data type) or result of a computation to a variable.
  • However, there are rules of naming variables which are given below.
    1. The variable names are case sensitive e.g., x and X are two different variable names.
    2. Variable names can contain letters (26 uppercase and 26 lowercase), digits (0∼9), and underscore (_) sign.
    3. The variable name must not start with a digit.
    4. The variable must not be a python keyword.
